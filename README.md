# open-auto-doc

Generate beautiful, AI-powered documentation websites from any GitHub repository in minutes.

Point it at your repos — frontend, backend, any language — and get a fully working [Fumadocs](https://fumadocs.dev) documentation site with architecture overviews, API references, component docs, data model docs, and getting started guides. All generated by AI that actually reads and understands your code.

## Features

- **One command** — `npx open-auto-doc` and you're done
- **GitHub Integration** — OAuth Device Flow, select from all your repos (public & private)
- **Language agnostic** — TypeScript, Python, Go, Rust, Java, Ruby, PHP, C#, and more
- **Deep AI analysis** — 4-stage pipeline: static parsing → architecture → detail extraction → synthesis
- **Multi-repo support** — Document multiple repos into a single unified site with sidebar navigation
- **CLAUDE.md aware** — Reads existing CLAUDE.md files from repos for richer, more accurate docs
- **Beautiful output** — Fumadocs + shadcn/ui with search, dark mode, responsive sidebar
- **You own the output** — Generates a standard Next.js project you can customize and deploy anywhere
- **BYOK** — Bring your own Anthropic API key

## Quick Start

### Prerequisites

- Node.js 18+
- A GitHub account
- An [Anthropic API key](https://console.anthropic.com/)

### Usage

```bash
npx open-auto-doc
```

The CLI walks you through:

1. **GitHub auth** — Opens your browser for OAuth Device Flow (no tokens to copy-paste)
2. **Repo selection** — Pick one or more repos from your account
3. **API key** — Enter your Anthropic key (saved locally for reuse)
4. **Analysis** — AI reads your codebase, understands the architecture, extracts docs
5. **Generation** — Outputs a complete Fumadocs site

Then start the dev server:

```bash
cd docs-site
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) to see your docs.

## Commands

| Command | Description |
|---------|-------------|
| `open-auto-doc init` | Full setup flow (default when running `npx open-auto-doc`) |
| `open-auto-doc init -o <dir>` | Specify output directory (default: `docs-site`) |
| `open-auto-doc generate` | Regenerate docs using saved `.autodocrc.json` config |
| `open-auto-doc login` | Authenticate with GitHub |
| `open-auto-doc logout` | Clear all stored credentials |

## What Gets Generated

### For each repository, AI analyzes and documents:

| Section | What it covers |
|---------|----------------|
| **Architecture Overview** | Tech stack, modules, data flow, entry points, key patterns |
| **Getting Started** | Prerequisites, installation, quick start, configuration |
| **API Reference** | Endpoints with methods, parameters, request/response bodies, auth |
| **Components** | UI components with props, usage examples, categories |
| **Data Models** | Schemas/models with fields, types, constraints, relationships |

## How It Works

### Analysis Pipeline

The analyzer runs a 4-stage pipeline on each repository:

1. **Static Parsing** — Builds file tree (respects `.gitignore`), detects languages from extensions, parses dependency files (`package.json`, `requirements.txt`, `go.mod`, `Cargo.toml`, `Gemfile`, `composer.json`), reads `CLAUDE.md` files
2. **Architecture Pass** — AI reads key files + CLAUDE.md context, produces high-level understanding of the codebase (tech stack, modules, data flow, entry points, patterns)
3. **Detail Pass** (runs in parallel) — Three specialized agents extract API endpoints, UI components, and data models concurrently
4. **Synthesis** — Combines all analysis into a getting started guide

### CLAUDE.md Integration

If your repo has a `CLAUDE.md` or `.claude/CLAUDE.md` file, its contents are injected into every AI prompt. This gives the AI insider knowledge about your project's conventions, architecture decisions, and important patterns — resulting in significantly better documentation.

## Configuration

### Credentials

Stored at `~/.open-auto-doc/credentials.json` (file permissions: `0600`).

Contains your GitHub OAuth token and Anthropic API key. Run `open-auto-doc logout` to clear.

### Re-generation

After the first run, a `.autodocrc.json` is saved in the output directory. Run `open-auto-doc generate` from that directory to re-analyze and regenerate docs without going through the full setup flow again.

### GitHub OAuth App

The CLI uses a default GitHub OAuth App. To use your own:

```bash
export OPEN_AUTO_DOC_GITHUB_CLIENT_ID=your_client_id
npx open-auto-doc
```

## Architecture

Monorepo with 4 packages:

```
packages/
├── cli/            # Published as "open-auto-doc" — Commander.js CLI
├── analyzer/       # AI code analysis engine — @anthropic-ai/sdk
├── generator/      # Handlebars MDX templates + Fumadocs site scaffolding
└── site-template/  # Next.js + Fumadocs + shadcn/ui template
```
